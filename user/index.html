User

Users are mywellness.com registered users that use technogym ecosystem tracking their activities, biometrics, manage theirs training program, challenges and training plan. Resources available on a User are: Profile, Aspiration, ActivitySummary.

Profile

Retreive the current logged user profile.

Url:
https://services.mywellness.com/api/[version]/User/[UserId]/Profile
HttpMethod:
POST

Parameters

access_token: string required
user access

Response

gender: string
M or F
culture: string
en-US (language-country)
timeZone: string

Aspiration

This request is used to retrieve last user aspiration map for the current authenticated user. Returns the user aspiration map.

Url:
https://services.mywellness.com/api/[version]/User/[UserId]/Aspiration
HttpMethod:
POST

Parameters

access_token: string required
user access

Response

gender: string
M or F
doneIn: string
facility data in which the aspiration has been taken. Can be omitted.
mainAspiration: string
[Power,Shape,Sport,Balance,Move,Fun]
secondaryAspiration: string
Can be omitted. [Power,Shape,Sport,Balance,Move,Fun]

Activity Summary

This request is used to retrieve current activity summary. Activity summary is composed by:

  • Training data: last training date, effectiveness and judice
  • Movement data: current weekly move and target
  • Current activity level (available only for mywellnesskey users)
Url:
https://services.mywellness.com/api/[version]/User/[UserId]/ActivitySummary
HttpMethod:
POST

Parameters

access_token: string required
user access

Response

lastTrainingEffectiveness: int
0 to 100
weeklyTargetMove: int
weeklyMove: int
