Mywellness cloud V1 API

Mywellness cloud V1 API is currently in BETA, therefore some parts are incomplete and inconsistent. Developers can expect regular changes and updates as we continue to iterate.

Access

All service call to mywellness cloud apis require an user access_token defining the user and the application making the call. Any registered application can obtain an user access token by first requesting an application sending an email to api@technogym.com, including a briefly descriptions about company / product / service and integration scope and purpouse.
Mywellness cloud team will contact the requestor in order to complete the application registration.
As registration completation result, application requestor receive an application client and secret using which he starts the OAuth authorization flow.
A mywellness cloud API application only has access to a user’s data after the user has authorized the application to use it.

Rate limit

Mywellness cloud API usage is limited on a per-application basis using a short term, 15 minute, limit and a long term, daily, limit.
The default rate limit allows 60 requests every 15 minutes, with up to 5.500 requests per day.
An application’s short term, 15 minute, limit is reset at natural 15 minute intervals corresponding to 0, 15, 30 and 45 minutes after the hour.
Requests exceeding the limit will return 403 Forbidden along with a JSON error message.

As an application grows its rate limit may need to be reassessed. To request an adjustment contact mywellness cloud team api@technogym.com.

Conventions

Response representation

Service response hase a standard defined format that has 2 main root fields named "data" and "errors". Those fields are mutually exclusive so only data or errors can be present.
