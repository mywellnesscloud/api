V1 API

The mywellness cloud V1 API is currently in BETA, therefore some parts are incomplete and inconsistent. Developers can expect regular changes and updates as we continue to iterate.

Getting Started

The mywellness cloud is an open platform that lets millions of people around the world track, measure and improve their health and fitness. The mywellness cloud API enables the exchange of select data in order to extend and augment the mywellness cloud user-experience and support complementary services in a collaborative wellness-ecosystem.
Three key principles govern the use of the mywellness cloud API: The data belongs to the user. We are stewards of the data and have an obligation to protect it and use it responsibly. Data can only be collected with explicit user permission. Only collect what you need and only use it as you say you will. User must have a mechanism to disable access and to request that any collected data be deleted. By visiting the Developer Portal, registering as a Developer and/or using the mywellness cloud API, you agree to abide by the terms and conditions set forth in the mywellness cloud API Terms of Use.

Access

All service call to the mywellness cloud API require an access_token defining the user and the application making the call. To obtain an access_token first you have to register you application at mywellness.com/appregistration, including a brief description about your company/ product / service and integration scope and purpose. Shortly after your request the mywellness cloud team will contact you to complete the registration.
See the Authentication page for more information about generating access tokens and the OAuth authorization flow.
Generally speaking a mywellness cloud API application only has access to a user's data after the user has authorized the application to use it.

Rate limiting

Mywellness cloud API usage is limited on a per-application basis using a short term, 15 minute, limit and a long term, daily, limit.
The default rate limit allows 60 requests every 15 minutes, with up to 5.500 requests per day.
An applicationâ€™s short term, 15 minute, limit is reset at natural 15 minute intervals corresponding to 0, 15, 30 and 45 minutes after the hour.
Requests exceeding the limit will return 403 Forbidden along with a JSON error message.

As an application grows its rate limit may need to be reassessed. To request an adjustment contact mywellness cloud team developer@technogym.com.

Conventions

Response representation

Service response has a standard defined format that has 2 main root fields named "data" and "errors". Those fields are mutually exclusive so only data or errors can be present. 

        {
           "data" : {},
           "errors": [{}]
        }
