Activity Upload

User's activities can be submitted from third parties by uploading them as files.

Currently the following file formats are supported (please keep in mind that .FIT is the preferred representation).

  • FIT
  • TCX
We have provided a working example of activities upload, the source code is available at this repository

Url:
https://services.mywellness.com/api/v1/User/[]UserId]/UploadActivity
HttpMethod:
POST

Parameters

token: string required
user access token
dataType: string required
FIT, TCX
data: string required
base64 encoded file 
		{
			"data": "TG9yZW0gaXBzdW0gZG9sb3Igc2l0IGFtZXQsIGNvbnNlY3RldHVyIGFkaXBpc2NpbmcgZWxpdCwgc2VkIGRvIGV==",
			"dataType": "FIT",
			"token": ""
		}

Response

created: bool
true if the activity has been successfully imported.
activityId: string
identifier of the newly created activity.
errorMessage: string
error message in case of failure. This field is null if activity creation is successful.		 
            {
              "data": {
                "results": [
                  {
                    "created": true,
                    "activityId": "03afc7a7-46c2-478f-9a8b-6ffcd1cd7305",
                    "errorMessage": ""
                  }
                ]
              },
              "version": "1.0"
            }
