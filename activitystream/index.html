Activity Stream

Activity Stream is the log of all interaction that user did using mywellness cloud applications. Stream entries can be retrieved per day or month. Currently only current and previous month data are available using apis. Each activity stream entity is available for a specific TimeFrame Id.
TimeframeId is an integer representation of a month like 201311 as november 2013 or for a specific day like 20131002 as 2 october 2013. Id must be included in resourse url.

Time Series

Retreive the user time and series data. Time and series data are available only for months for these categories: Calories, Move, Clycling distance, Running Distance and Time.
Response always contains all month days also if user has no results.

Url:
https://services.mywellness.com/api/[version]/ActivityStream/[TimeFrameId]/TimeSeries
HttpMethod:
GET

Parameters

token: string required
user access token
dataType: string required
Move, Calories, RunningDistance, CyclingDistance 
            {
               "dataType": "Move",
               "token": ""
            }

Response

day: int
integer representation of a day yyyyMMdd
value: string
counter representation as string. In case of distance the data is represented in meter
rawValue: double
counter raw value. In case of distance the data is represented in meter

Daily Counters

This request is used to retrieve a complete daily user activity counters. Response is organized in Technogym Cerfied data and totals.
Technogym Certified data are data that are collected on a technogym equipment using a tgs/mywellness key device or using a connected equipment. Doing so Techngnogym certifies that these activities has been performed by the user. Technogym Certified exercises are not modifiable so are stored exactly how the user did them on an equipment. Total contains all activities user did manually or automatically.

Url:
https://services.mywellness.com/api/[version]/ActivityStream/[TimeFrameId]/DailyCounters
HttpMethod:
POST

Parameters

token: string required
user access		 
            { 
              "token" :""
            }

Response

day: int
integer representation of a day yyyyMMdd
technogymCertified: object
Technogym Certified user data.
duration: int
activities total duration in seconds.
move: int
activities total move
calories: int
activities total calories
runningDistance: int
activities total run distance in meter
cyclingDistance: int
activities total cycling distance in meter		 
           {
            "data": {

                "technogymCertified": {
                  "duration": 0,
                  "move": 0,
                  "calories": 0,
                  "runningDistance": 0,
                  "cyclingDistance": 0
                },
                "totals": {
                  "duration": 0,
                  "move": 0,
                  "calories": 0,
                  "runningDistance": 0,
                  "cyclingDistance": 0
                }
              },
              "version": "1.0"
            }

Logs

This services provide all user activities logbook. This contains logs of all iteration user did on mywellness cloud. Logs can be retrieved per day or month depending on the [TimeFrameId].

Url:
https://services.mywellness.com/api/[version]/ActivityStream/[TimeFrameId]/Logs
HttpMethod:
POST

user access
